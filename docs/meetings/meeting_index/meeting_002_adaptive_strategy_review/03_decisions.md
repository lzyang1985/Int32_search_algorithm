# 决议记录：自适应策略评审会

## 总决议

**会议一致确认：方案 A（AVX2 SIMD 向量化二分查找）为唯一查询路径。不引入任何形式的多路径自适应策略。**

meeting_001 决议 D-008 和 D-009 的结论经本次会议复核后，仍然成立且无修改必要。

---

## 逐项决议

### 决议 D-010：拒绝自适应多路径

**决议**：不引入基于 high16 directory 桶大小判断的自适应策略（小桶 lo16 扫描 / 大桶 AVX2 二分 / 极端大桶 mid8 二级目录）。

**理由**：

1. **性能倒退**：在 5M-10M 最常见场景下，自适应路径实质退化到 B2（high16dir + AVX2 二分），实测比纯 A 慢 25%-56%。
2. **dir 税无法摊销**：high16 directory 的 O(1) 定位开销（~16-20 cy/查询）在桶均值 > T_small 时是每位查询者的净亏损。
3. **B1 的优势不相关**：1M 数据下 B1 的 2.1x（75 vs 154 cy）绝对差距仅 22ns，且 1M 不在目标千万级场景内。
4. **工程代价过高**：+150 行代码（2.3x）、+20MB 内存（+50%）、3 个代码路径（3x 测试面）、2 个调优阈值。
5. **安全风险**：high16 dir 构建存在 `prev_h=0xFFFF` 初始化 bug（静默假阴性），三数组 COW 若未做 struct 级原子交换会产生跨代数据不一致。
6. **与 D-003 冲突**：D-003 全票决议"不再使用低 N 位作为 key"，自适应方案实质回到分位策略（high16 桶号 + lo16 桶内 key）。
7. **阻断未来扩展**：自适应决策树与 AVX-512/布隆/桶排序组合会产生 O(2^n) 测试爆炸。

**表决**：5/5 一致通过。

---

### 决议 D-011：AVX2 SIMD 二分 = 唯一查询路径

**决议**：重申 meeting_001 决议 D-008，方案 A（AVX2 SIMD 向量化二分查找）为最终且唯一的查询算法路径。

**确认项**：
- 数据结构：`int32_t vals[n]`（排序数组），无额外索引
- 查询算法：8 路块状 AVX2 二分 → 尾部退化为标量二分
- 内存（10M）：40MB
- 性能（10M uniform）：~172 cycles/query（5.1x vs 标量二分）
- 数据分布敏感度：0.3%（完全不敏感）

**表决**：5/5 一致通过。

---

### 决议 D-012：已知 Trade-off 记录

**决议**：B1（high16dir + lo16 扫描）在 1M 数据时 75 cy（2.1x vs 方案 A 的 154 cy）的已知优势，记录为不做特殊处理的已接受 trade-off。

**理由**：
- 绝对差距仅 22ns（~3.5GHz）
- 1M（40MB）不在目标千万级场景内
- 方案 A 的 154 cy（44ns）已足够快
- 为优化此边缘场景付出的复杂度不值得

**未来保留**：若有强烈的小数据量优化需求，考虑实现为模块级分发的独立变体（`n <= 2M` 条件），而非嵌入核心搜索循环的自适应逻辑。

**表决**：5/5 一致通过。

**决议**：meeting_001 和 meeting_002 已解决所有技术分歧。方案 A 确认为不可撼动的最优解。终止 POC 阶段，进入立项工作流（Align → Architect → Atomize → Approve）。

**表决**：5/5 一致通过。

## 第三轮决议（人工修正方案：构建时一次性选路）

### 决议 D-014：批准构建时一次性选路方案

**决议**：以构建时一次性选路方案替代原先的纯方案 A 单一路径。方案 A（纯 AVX2 二分）保持为默认/回退路径，B1（high16 dir + lo16 SIMD 扫描）作为数据均匀时自动选中的优化路径。

**与 D-010 的关系**：D-010 否决的是"每查询自适应多路径"（dir 税每查询支付、热路径分支、组合爆炸）。本方案为构建时一次性选路（dir 仅在构建时临时使用，热路径零开销），与 D-010 不冲突。

**理由**：
1. 构建时选路与查询时自适应有架构本质区别——前者热路径零开销，后者每查询支付 dir 税
2. 1M-2M 均匀数据下 B1 有 1.5x-2.1x 的实测加速，且数据量在目标百万级范围内
3. 最差退化路径 = A（172 cy），不会出现旧方案中 B2 慢 56% 的系统性退化
4. 新增代码 ~102 行（1.85x），两条完整路径正交解耦，各自独立演进
5. B1 内存增量自限在 ≤ 8.4MB（仅在 ≤ 4M 时触发），运行时可忽略

**表决**：5/5 通过。

### 决议 D-015：分布检测与阈值规则（D-019 POC 修正后）

**决议**：采用以下规则判断 B1 还是 A 路径：

```
1. 构建 high16 directory dir[65537]
2. 一致性校验：dir[0]==0, dir[65536]==n, dir 单调不递减。失败 → PATH_A
3. 扫描计算 max_sz = max(dir[i+1] - dir[i])
4. 决策：
     IF max_sz > 0.1 × n      → PATH_A  （检测倾斜分布，B1 退化 7.4x）
     IF max_sz ≤ 150          → PATH_B1 （小桶场景，B1 实测胜）
     ELSE                     → PATH_A  （回退）
```

**D-019 POC 修正说明**（原版本见下方"旧版 D-015"）：

原规则以 `weighted_avg ≤ 45` 为指标。D-019 POC v3 实测发现：1.5M 均匀数据下 weighted_avg=92.6（远超 45），但 B1 以 0.89x 赢 A（120 vs 135 cy）。`weighted_avg` 的 Poisson 理论模型与实测差 ~4x，不可靠。

修正为 `max_bucket ≤ 150`——直接测量"最坏桶扫描成本"，4 个测试点全对：

| N | max_bucket | ≤150? | B1 vs A | 判定 |
|---|-----------|-------|---------|------|
| 1.5M | 135 | ✅ | B1 0.89x | PATH_B1 ✅ |
| 2.0M | 169 | ❌ | A 1.18x | PATH_A ✅ |
| 2.5M | 199 | ❌ | A 1.48x | PATH_A ✅ |
| 3.0M | 248 | ❌ | A 2.13x | PATH_A ✅ |

Crossover 点：~1.6M（max_bucket ≈ 150）。参见 [07_poc_benchmark_v3_report.md](07_poc_benchmark_v3_report.md)。

**表决**：修正通过。

### 决议 D-016：prev_h Bug 修复为前置条件

**决议**：`high16_dir_build()` 中 [poc_benchmark_v2.c:L80](file:///c:/Users/Administrator/Documents/trae_projects/Int32_search_algorithm/src/poc_benchmark_v2.c#L80) 的 `prev_h = 0xFFFF` 初始化 bug 必须在正式实现中修复。修复方式：使用标记 `int first = 1` 处理首个元素，或改用 `prev_h = 0xFFFE`（不在 int32 的 high16 值域内）。

**附加要求**：
- 构建后必须做一致性校验：`dir[0] == 0, dir[65536] == n, dir` 单调不递减
- 校验失败 → 回退到 A 路径，不启用 B1

**严重程度**：此 bug 在 1M 均匀数据下触发概率极高（期望 15 个元素触发）。无此修复，B1 路径不可靠。

**表决**：5/5 通过。

### 决议 D-017：COW 一致性保障

**决议**：B1 路径的 vals + lo16 + dir 三个数组必须封装在单个 struct 中，COW 更新时对该 struct 指针做原子交换，而非逐个指针交换。

```c
typedef struct {
    const int32_t  *vals;
    const uint16_t *lo16;
    const int32_t  *dir;
    size_t          n;
} b1_snapshot_t;
// 原子交换: atomic_store(&g_snapshot, new_snapshot);
```

A 路径单数组天然不需要此要求。

**表决**：5/5 通过。

### 决议 D-018：D-003 条件豁免

**决议**：在 meeting_001 决议 D-003（"禁止使用低 N 位作为 key"）中增加条件豁免条款：B1 路径（high16 桶号 + lo16 桶内 key）仅在构建时分布测量验证数据均匀性（max_sz ≤ 0.1×n 且 max_bucket ≤ 150）后才启用。不满足条件时回退到 D-003 兼容的纯 AVX2 二分路径。

**此豁免的性质**：精细化（refinement），非违背。D-003 的核心关切是"分位策略的可靠性依赖数据分布，而分布不可假设"。本方案不再假设——我们在测量。测量通过才启用，不通过则回退。

**表决**：5/5 通过。
