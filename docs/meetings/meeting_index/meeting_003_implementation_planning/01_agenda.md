# 议程：实施方案讨论会

## 议题列表

### 议题 1：模块划分

基于已确定的技术方案全貌（A+B1 双路径、构建时一次性选路），将完整系统划分为独立模块：

- **核心数据层**：排序数组、构建-查询分离、COW 更新
- **查询引擎层**：AVX2 SIMD 向量化二分（A 路径）、high16 dir + lo16 SIMD 扫描（B1 路径）
- **构建/选路层**：数据排序、分布分析（dir 构建 + 一致性校验 + max_bucket 决策）
- **平台适配层**：SIMD 多版本（标量/SSE2/AVX2/AVX-512）、线程（pthread/Win32）、内存对齐
- **可选扩展层**：布隆过滤器（#ifdef USE_BLOOM_FILTER）、桶排序预处理
- **公开 API 层**：不透明句柄、错误码、内存所有权模型

### 议题 2：子项目拆分

是否需要将整个项目拆分为多个可独立编译/测试的子项目？例如：

- 子项目 1：核心库（libint32search）—— 公开 API + 核心引擎
- 子项目 2：Benchmark 工具 —— 性能回归测试
- 子项目 3：测试套件 —— 单元测试 + 集成测试 + 压力测试
- 子项目 4：文档与示例

### 议题 3：子项目实施顺序

确定各模块/子项目的依赖关系和实施优先级：

- 关键路径识别
- 依赖关系分析
- P0/P1/P2 优先级划分
- MVP（最小可运行版本）范围定义
