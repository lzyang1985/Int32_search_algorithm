# 总需求文档 — Int32 查找算法库

## 1. 项目概述

### 1.1 项目名称

**Int32 查找算法库**（libint32search）

### 1.2 项目目标

开发一个高性能 C 语言库，提供 Int32 数据的精确查找能力。核心目标是利用 SIMD（AVX2）指令集实现远超标准二分查找的查询速度。

### 1.3 项目背景

经过两场需求评审会议和三轮 POC benchmark 验证，确认了以下技术路线的可行性：
- 排序数组 + AVX2 SIMD 向量化二分查找，在 1000 万数据规模下达到 **3.5x-5.1x** 加速比（vs 标量二分）
- 构建时一次性选路（A+B1 双路径），热路径零开销

---

## 2. 功能需求

### 2.1 核心功能

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| FR-01 | 批量数据构建 | P0 | 输入 Int32 数组，内部排序后构建查询索引 |
| FR-02 | 精确查找 | P0 | 给定 Int32 key，返回第一个匹配元素的索引 |
| FR-03 | 资源销毁 | P0 | 释放构建阶段分配的所有内存 |
| FR-04 | 数据重建 | P1 | 支持 COW 方式替换内部数据集（不阻塞查询） |

### 2.2 扩展功能

| 编号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| FR-05 | 范围查询接口 | P2 | 预留接口声明，标注 `reserved`，不实现 |
| FR-06 | 布隆过滤器预筛 | P2 | `#ifdef USE_BLOOM_FILTER` 编译开关，减少不命中查询开销 |

### 2.3 已人工确认的关键决策（meeting_001 Q1-Q3）

| 决策 | 确认结果 | 影响 |
|------|----------|------|
| Q1 动态增删 | 偶尔少量添加删除 | 以批量重建为主，不需要在线增删的高复杂度方案 |
| Q2 范围查询 | 保留接口，不实现 | 接口预留，排序数组天然支持范围查询 |
| Q3 AVX 加速 | 速度为王 | SIMD/AVX 全面投入，适当内存和复杂度换取性能 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 验证方式 |
|------|--------|----------|
| 查询延迟（10M uniform, 50% hit） | < 200 cycles/query | Benchmark 回归 |
| 内存占用（10M 数据） | ≤ 40 MB（Path A）/ ≤ 84 MB（Path B1） | 静态分析 |
| SIMD 加速比（vs 标量二分） | ≥ 3.5x @ 10M | Benchmark 回归 |
| 数据分布敏感度 | < 5% 方差 | 多分布 benchmark |

### 3.2 安全需求

| 编号 | 需求 | 说明 |
|------|------|------|
| SR-01 | SIMD 边界安全 | n=0~64 所有值不越界读取 |
| SR-02 | COW 内存序 | B1 路径三数组 struct 级原子交换，release/acquire 语义 |
| SR-03 | dir 一致性校验 | 构建阶段强制校验，失败回退 Path A（生产构建也保留） |
| SR-04 | 内存泄漏防护 | create 失败时资源回滚；destroy 幂等性 |
| SR-05 | ASan/UBSan 零告警 | 开发阶段默认开启 sanitizer |

### 3.3 兼容性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| CR-01 | 编译器 | GCC ≥ 8.0（主力），MSVC + Clang 通过 CMake 支持 |
| CR-02 | 平台 | Linux（主力），Windows（Phase 3） |
| CR-03 | C 标准 | C11（`_Alignas`、`_Generic`、`stdatomic.h`） |
| CR-04 | 指令集 | 运行时 CPUID 检测，编译时多版本（标量→SSE2→AVX2→AVX-512） |

### 3.4 工程需求

| 编号 | 需求 | 说明 |
|------|------|------|
| ER-01 | 构建系统 | Makefile（主力）+ CMakeLists.txt（辅助）+ README.txt（gcc 命令） |
| ER-02 | 命名规范 | 下划线命名法（snake_case） |
| ER-03 | API 风格 | 不透明句柄 `void*`，`int` 错误码，调用方管理内存 |
| ER-04 | 代码量 | 预估 ~1000 行（不含 POC 归档） |

---

## 4. 约束条件

| 约束 | 说明 |
|------|------|
| 语言 | 纯 C（C11），不引入 C++ |
| 依赖 | 最小化：仅 xxHash（已有）+ 标准库 |
| 线程 | 构建-查询分离 + COW，查询路径零锁 |
| 内存 | 调用方分配输出缓冲区，库不跨边界分配 |
| API | 单一公开头文件 `int32_search.h`，内部结构体不暴露 |

## 5. 三阶段交付计划（meeting_003 D-023）

```
Phase 1: MVP — AVX2 二分（Path A 单路径）
  ├── 功能：FR-01 构建、FR-02 查找、FR-03 销毁
  ├── 查询：AVX2 8 路块状 SIMD 二分 + 标量二分回退
  ├── 性能：10M ~172 cy/query（5.1x vs 标量）
  └── 不含：B1 路径、COW、布隆、AVX-512

Phase 1.5: v0.2 — 多线程（Path A COW）
  ├── 新增：FR-04 数据重建（Path A COW，原子指针交换）
  ├── 新增：platform_thread.c（原子操作封装）
  └── 验收：ThreadSanitizer 零告警

Phase 2: v1.0 — A+B1 双路径
  ├── 新增：B1 路径（high16 dir + lo16 SIMD 扫描）
  ├── 新增：D-015 分布分析 + 自动选路
  ├── 新增：B1 COW（struct 级三指针原子交换）
  └── 性能：1M 数据 B1 ~75 cy/query（2.1x vs A）

Phase 3: v1.1 — 扩展与跨平台
  ├── FR-05 范围查询接口（预留）
  ├── FR-06 布隆过滤器
  ├── SSE2 / AVX-512 编译版本
  └── Windows 平台移植
```

## 6. 验收标准

### 6.1 Phase 1 MVP 验收

- [ ] `gcc -O3 -std=c11 -mavx2` 编译通过，零警告
- [ ] API 符合不透明句柄 + 错误码 + 内存所有权模型
- [ ] 正确性：100 万次随机查询结果与 `bsearch()` 完全一致
- [ ] 性能：10M uniform 50% 命中率 < 200 cy/query
- [ ] 安全：`-fsanitize=address,undefined` 编译零告警
- [ ] SIMD 边界：n=0~64 所有值测试通过

### 6.2 Phase 1.5 多线程验收

- [ ] Path A COW 原子指针交换：`_Atomic const int32_t*` 正确同步
- [ ] `int32_search_rebuild()` 不阻塞并发查询
- [ ] ThreadSanitizer 并发压力测试零告警
- [ ] 旧版本延迟回收无内存泄漏

### 6.3 Phase 2 v1.0 验收

- [ ] A vs B1 交叉验证 100 万次结果逐位一致
- [ ] B1 COW struct 级三指针原子交换正确（D-017）
- [ ] B1 路径 ThreadSanitizer 零告警
- [ ] 倾斜数据自动回退 Path A（max_sz > 0.1×n 检测）
- [ ] 1.5M 均匀数据自动选中 B1（max_bucket ≤ 2000）
